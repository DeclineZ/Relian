2. Install dependencies:

3. (Optional) Configure API endpoint for dev:
- The app uses import.meta.env.VITE_API_URL when running in the browser (not inside Capacitor). To set it locally, create a .env file in the project root containing:

7. Troubleshooting
--------------------------------------------------------------------------------

AI or backend calls failing
- If axios calls to /api fail in browser, ensure VITE_API_URL is set and CORS is allowed on backend, or rely on Vite proxy (vite.config.js) during dev.
- When running in Capacitor, the code uses a hard-coded BASE pointing at the Railway URL. Change that if you run your own backend or want local dev.

Decks not updating or missing
- LocalStorage may have stale data. Clear localStorage 'decks' key to re-seed from src/data/decks.json.

Mobile native issues
- Run npx cap sync after web build to ensure native projects contain latest web bundle.
- Ensure correct Capacitor CLI and native SDK versions.

Debugging tips
- Use React DevTools to inspect DeckContext.Provider values (decks, userXP)
- Add console.log inside DeckContext and FlashcardPlayer for tracing state updates
